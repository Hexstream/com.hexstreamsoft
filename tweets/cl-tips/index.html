Standard library (and style suggestions) (20) ⚓

Tip: Don't write:(let ((*print-pretty* nil)) (write object))Write this instead:(write object :pretty nil)Also :base :case :circle etc.

— Jean-PhilippeParadis (@HexstreamSoft) July 20, 2012

Tip: Don't write (format nil "~S" object), write (prin1-to-string object) instead. There's also WRITE-TO-STRING and PRINC-TO-STRING.

— Jean-PhilippeParadis (@HexstreamSoft) July 20, 2012

Pathname for current .lisp file: #.(or *compile-file-truename* *load-truename*) Works reliably with LOAD/COMPILE-FILE/ASDF/Emacs/Slime/fasl.

— Jean-PhilippeParadis (@HexstreamSoft) July 20, 2012

My nomination for most obscure CL feature hidden in plain sight: WITH-INPUT-FROM-STRING's INDEX argument. Somewhat complicated semantics.

— Jean-PhilippeParadis (@HexstreamSoft) July 27, 2012

CL tip: Most string comparison operators (STRING/=, STRING-NOT-EQUAL, STRING<, etc) return a mismatch-index, not just a generalized boolean!

— Jean-PhilippeParadis (@HexstreamSoft) July 29, 2012

SUBTYPEP has interesting return values: 2 booleans meaning "Yes, definitely", "Definitely not" and "I'm not sure exactly, let's assume not."

— Jean-PhilippeParadis (@HexstreamSoft) July 29, 2012

CL tip: MEMBER returns a tail of the list. How to get the "head"? (let ((list '(a b 7 c))) (ldiff list (member-if #'numberp list))) => (A B)

— Jean-PhilippeParadis (@HexstreamSoft) August 7, 2012

I used to write (and (consp foo) (eq foo 'symbol)). Now I prefer (typep foo '(cons (eql symbol))): more direct, "one" conceptual operation.

— Jean-PhilippeParadis (@HexstreamSoft) August 24, 2012

Nomination for most little-used feature in CL: ,. is to ,@ as NCONC is to APPEND. With reason: hard to reason about. lispworks.com/

— Jean-PhilippeParadis (@HexstreamSoft) September 3, 2012

Used to write (let ((v i)) x y v) but now prefer (let ((v i)) (prog1 v x y)): states intent up-front; won't forget V return while writing.

— Jean-PhilippeParadis (@HexstreamSoft) September 5, 2012

So (let ((v i)) (prog1 v x y)) is better for human readers/writers, especially as "x y" get more complex. One downside: extra indentation.

— Jean-PhilippeParadis (@HexstreamSoft) September 5, 2012

CL tip: Exclusive bounds for compound type specifiers: (integer 0 (10)) == (integer 0 9). Useful for '(integer 0 (#.multiple-values-limit)).

— Jean-PhilippeParadis (@HexstreamSoft) September 10, 2012

(let ((box (make-array '() :initial-element 'e))) (values (aref box) (array-total-size box))) => E, 1. Same reason that (apply #'* '()) => 1

— Jean-PhilippeParadis (@HexstreamSoft) September 18, 2012

CL tip: Lil'-known array functions: ARRAY-TOTAL-SIZE, ARRAY-RANK, ARRAY-DIMENSION, ROW-MAJOR-AREF, ARRAY-ROW-MAJOR-INDEX, ARRAY-IN-BOUNDS-P.

— Jean-PhilippeParadis (@HexstreamSoft) September 19, 2012

CL tip: All rounding functions ([F]TRUNCATE, [F]ROUND, [F]FLOOR, [F]CEILING) accept an optional DIVISOR argument. (floor 26 6) => 4, 2

— Jean-PhilippeParadis (@HexstreamSoft) September 19, 2012

My nominations for most glaring X3J13 concessions: CASE's OTHERWISE (does anyone use it?) and PAIRLIS' non-commitment to order of new pairs.

— Jean-PhilippeParadis (@HexstreamSoft) September 19, 2012

CL tip: Continually confused about (sort seq #'<) VS (sort seq #'>)? Just remember that (apply #'< (sort list #'<)) always returns true.

— Jean-PhilippeParadis (@HexstreamSoft) September 26, 2012
"Unless SEQ has duplicates", as pointed out by Greg Pfeil in a reply.

CL tip: Vector of N random integers below L: (map-into (make-array N) (lambda () (random L))) (also MAKE-LIST, MAKE-SEQUENCE, MAKE-STRING)

— Jean-PhilippeParadis (@HexstreamSoft) September 28, 2012

CL tip: Result of (gensym "VAR") might print with escapes, depending on (readtable-case *readtable*). (gensym (string '#:var)) averts this.

— Jean-PhilippeParadis (@HexstreamSoft) September 28, 2012

CL tip: (values #5(1 2 3) (array-dimensions #2a((a1 a2) (b1 b2) (c1 c2)))) => #(1 2 3 3 3), (3 2) lispworks.com/ lispworks.com/

— Jean-PhilippeParadis (@HexstreamSoft) September 29, 2012

My nomination for most complex (though justified) syntax in CL: DEFINE-METHOD-COMBINATION. It's truly breathtaking! lispworks.com/

Lambda lists (3) ⚓

My nomination for weirdest CL feature: pass :allow-other-keys t in any &key args context to disable keyword args check. lispworks.com/

— Jean-PhilippeParadis (@HexstreamSoft) July 23, 2012

Avoid using &optional in public APIs. &key is more future-proof and makes for more explicit calls; &optional and &key don't mix very well.

— Jean-PhilippeParadis (@HexstreamSoft) July 26, 2012

&aux does have one tremendously practical utility: saving one indentation level. Also for BOA lambda lists but I don't really use DEFSTRUCT.

Multiple values (12) ⚓

(multiple-value-bind (a b c) (foo) (bar)) == (multiple-value-call (lambda (&optional a b c &rest rest) (declare (ignore rest)) (bar)) (foo))

— Jean-PhilippeParadis (@HexstreamSoft) July 20, 2012

(apply #'foo bar (multiple-value-list (baz))) is ugly and inefficient. Do (multiple-value-call #'foo bar (baz)) instead.

— Jean-PhilippeParadis (@HexstreamSoft) July 21, 2012

MULTIPLE-VALUE-CALL helped me understand continuation-passing-style: returning values == calling continuation with those values.

— Jean-PhilippeParadis (@HexstreamSoft) July 22, 2012

MULTIPLE-VALUE-CALL also showed me how funargs and retvals are such close duals of each other. Future dialect: take retvals more seriously.

— Jean-PhilippeParadis (@HexstreamSoft) July 22, 2012

I used to write (let ((a (if c x y)) (b (if c foo bar)))). Now I prefer (multiple-value-bind (a b) (if c (values x foo) (values y bar))).

— Jean-PhilippeParadis (@HexstreamSoft) August 28, 2012

Some people may not realize how frigging efficient multiple-value returns/binding really is. This might explain in part why it's underused.

— Jean-PhilippeParadis (@HexstreamSoft) August 29, 2012

CL tip: Indentation-saving trick: (multiple-value-bind (a1 a2 b c1 c2 c3) (multiple-value-call #'values (values 1 2) 3 (values 4 5 6)) ...)

— Jean-PhilippeParadis (@HexstreamSoft) September 9, 2012

I sometimes find fruitful to indicate a function's return values similarly to its arguments: (defun foo (a1 a2) '(values v1 v2 v3 v4) ...).

— Jean-PhilippeParadis (@HexstreamSoft) September 10, 2012

CL tip: Returning primary value: (values form) is popular, but as FORM gets bigger retvals count is obscured. (identity form) more explicit.

— Jean-PhilippeParadis (@HexstreamSoft) September 27, 2012

CL tip: (lambda (f a &optional (b nil ?)) (if ? (funcall f a b) (funcall f a))) ==> (lambda ... (multiple-value-call f a (if ? b (values))))

— Jean-PhilippeParadis (@HexstreamSoft) September 28, 2012

(flet ((f (v) ...)) (let ((a (f x)) (b (f y))) BB)) ==> (multiple-value-bind (a b) (flet ((f (v) ...)) (values (f x) (f y))) BB) = f.scope--

— Jean-PhilippeParadis (@HexstreamSoft) October 1, 2012

multiple-value-&bind fixes MULTIPLE-VALUE-BIND's obliviousness to &key and &rest multival returns (otherwise awkward): github.com/Hexstream/

— Jean-PhilippeParadis (@HexstreamSoft) October 1, 2012

(defun %make-accumulator () '(values collect finish) (let ((list nil)) (values (lambda (o) (push o list)) (lambda () (nreverse list))))))

— Jean-PhilippeParadis (@HexstreamSoft) October 2, 2012

Classes (6) ⚓

CL tip: I like to prefix all private class slots with % to be more explicit and avoid accidental "leaking": (defclass foo () (%function))

— Jean-PhilippeParadis (@HexstreamSoft) September 29, 2012

CL tip: (defclass foo () ((%bar :reader foo-bar :accessor %foo-bar))) avoids "leaking" a writer while retaining convenience for implementor.

— Jean-PhilippeParadis (@HexstreamSoft) September 29, 2012

CL tip: Want to (effectively) override an initform in a subclass? (defclass subclass (superclass) () (:default-initargs :init 'new-default))

— Jean-PhilippeParadis (@HexstreamSoft) September 29, 2012

CL tip: :default-initargs often a better idea than :initform, especially when auxiliary methods are defined on INITIALIZE-INSTANCE and such.

— Jean-PhilippeParadis (@HexstreamSoft) September 29, 2012

CL tip: (defmethod initialize-instance :around ((o c) &rest r &key k &allow-other-keys) (apply #'call-next-method o :k (canonicalize k) r))

— Jean-PhilippeParadis (@HexstreamSoft) September 29, 2012

CL tip: Each of :initarg, :reader, :writer and :accessor can be specified multiple times for a single class slot. lispworks.com/

— Jean-PhilippeParadis (@HexstreamSoft) September 29, 2012

Macros (11) ⚓

CL DSL tip: Don't build thick macro layers. Instead, 1. solid foundation of functions and objects 2. thin layer of syntax sugar with macros.

— Jean-PhilippeParadis (@HexstreamSoft) July 23, 2012

CL advice: It's almost never a good idea to generate user-visible symbols as part of macroexpansions. Make user pass a symbol explicitly.

— Jean-PhilippeParadis (@HexstreamSoft) July 25, 2012

CL advice: Don't wrap DEFCLASS to remove the explicitness. Gaining more experience with better OO designs will reveal erroneous assumptions.

— Jean-PhilippeParadis (@HexstreamSoft) July 25, 2012

Key to understanding setfable places: DEFINE-SETF-EXPANDER/GET-SETF-EXPANSION and CLHS 5.1. DEFSETF/DEFINE-MODIFY-MACRO mere conveniences.

— Jean-PhilippeParadis (@HexstreamSoft) July 25, 2012

Often overlooked macro quality issue: correct handling of all declarations usually doesn't happen by accident, you'll often have to parse.

— Jean-PhilippeParadis (@HexstreamSoft) July 31, 2012

I dislike macros that expand to calls to themselves, as interactive macroexpansion might not reveal all the work of the macro at once.

— Jean-PhilippeParadis (@HexstreamSoft) July 31, 2012

Interactive macroexpansion of calls to MACROLET-defined macros is not commonly supported, so it's sometimes more pragmatic to use DEFMACRO.

— Jean-PhilippeParadis (@HexstreamSoft) July 31, 2012

Macro design: Full code walking implies full macroexpansion of all macros, which almost guarantees an ugly, unreadable MACROEXPAND result.

— Jean-PhilippeParadis (@HexstreamSoft) October 1, 2012

Macro design: Full code walking is too conceptually and implementationally complex. Redesign semantics with FLET/MACROLET/SYMBOL-MACROLET.

— Jean-PhilippeParadis (@HexstreamSoft) October 1, 2012

New library: MACRO-LEVEL saves indentation; more concise and direct. (macro-level ...) == (macrolet ((m () ...)) (m)) hexstreamsoft.com/projects/

— Jean-PhilippeParadis (@HexstreamSoft) October 3, 2012
