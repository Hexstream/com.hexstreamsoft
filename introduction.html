Introduction

Hello! I have 6 intensive years of Common Lisp experience.

Hello, I'm Jean-Philippe Paradis and for the last ~6 years I was (and still am) intensively in Common Lisp development. During that time, I developed many very useful, novel and non-trivial libraries (notably, a big bunch of libraries which constitute a kind of web framework). I recently open-sourced (MIT license) almost 100% of my code on Github.

Unfortunately, since I was the only user of my libraries, there are several "quality issues" I absolutely didn't worry about until the open-sourcing, notably they're completely undocumented. As a result, almost all of them are, though publicly available, currently unsuitable for public consumption unless you're some kind of masochist looking for a great way to waste his time.

I have recently embarked on a long journey (at least 1 year) to thoroughly cleanup and comprehensively document all my libraries so that anyone can use them easily. I know this will be a ton of work but I am determined to do it, I'm sick of being the only user of my innovations.

(Don't get fooled by this rudimentary hand-coded page (and others), I made "really nice" HTML and CSS libraries and have some good experience with web development, but these web libraries currently are in a very sorry state so I'll use them again when their cleaning up is done and I don't want to get carried away with making high-quality pages for just one or two projects.)
